hexo修改文章，分类和标签

修改文章的内容(包括时间，标题，文件名)，和修改分类与标签一样，直接对文章修改即可，对本地和远程发布同样时自动生效，执行下列命令，之前发布的内容都会被自动替换。

hexo文章删除

  • 本地删除

    进入source/_post文件夹下，找到要删除的文件，手动删除。

    执行本地删除 hexo cl && hexo g

    清理本地缓存，将生成博客系统所有文件放到public文件夹下，执行 hexo s 即可本地查看文件已经被删除。

  • 已发布文章删除

    如果文章已经使用git发布，此时会在本地创建一个名为 .deploy_git的文件夹。使用 hexo cl && hexo g 命令后，继续使用 hexo d 命令，会导致public文件夹中的内容被拷贝到.deploy_git文件夹中，并把该文件夹里的所有文件全部push到远程库，此时本地删除的文件仍然在.deploy_git文件夹中保存，故需要手动删除.deploy_git文件夹。【来自参考】

删除多余的标签和分类

实际上，不需要使用命令手动创建分类和标签，第一次创建完标签和分类后，后面写文章时只需在页面带上分类和标签，对本地发布和远程发布，就可以自动创建指定分类和标签。

如果多次执行 hexo new page "xxx"（”xxx”代表任意且不同的多个名字），会导致根目录下source文件夹下产生多个文件夹，同样的，垃圾文件，手动删除即可。
