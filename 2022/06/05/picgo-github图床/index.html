picgo github图床

Posted on 2022-06-05 Edited on 2022-06-15 In major , website , pic_bed Views: Comments: Symbols count in article: 1.8k Reading time ≈ 2 mins.

声明：本次内容仅用作记录一次CDN加速实验，不运用于任何实际生产环境之中。

目前无法使用jsDelivr对GitHub图床加速，之前的jsDelivr ICP吊销事件已经说明了原因，但是更换cdn加速，或许有所帮助，鉴于GitHub本身的巨大的优势，记录下这一次实验案例。

搭建

安装picgo，vscode，vscode picgo插件

参考我之前的文章 picgo-sm-ms-v2-图床搭建

创建GitHub仓库

假设你已经完成gitub账号注册，新建一个图床仓库：

名称 picgo_imgs
属性 public
分支 main
其它随意

GitHub生成access_token:

点击 Github右上角的用户头像→Settings→Developer settiings→Personal access tokens→Generate new token，在Note一栏输入创建的仓库名称，并勾选repo，最后在最下方点击Generate token按钮，就生成了对应的Token。

note: 注意这个 Token 只会显示一次，自己先保存下来，或者等后面配置好 PicGo 后再关闭此网页

配置PicGo

设定仓库名（Repo）：按照 用户名/图床仓库名 的格式填写

设定分支名（Branch）：从2020.10.01开始，github的默认分支名变更为main

设定 Token：粘贴之前生成的 Token

指定存储路径（Path）：填写想要储存的路径，如picgo/imgs/，所有通过插件上传的图片都在图床仓库中的picgo/imgs文件夹下（后面的/必须加上，不然imgs就是上传后的图片名前缀）

设定自定义域名（Custom Url）：它的的作用是，在图片上传后，PicGo 会按照自定义域名 上传的图片名的方式生成访问链接，放到粘贴板上，因为我们要使用 jsDelivr CDN 加速访问，所以可以设置为：

https://cdn.jsdelivr.net/gh/[用户名]/[仓库名]@[分支名]

note: 这里使用了jsDelivr进行加速，原生方式可以不填，默认使用GitHub自定义域名：

https://raw.githubusercontent.com/[用户名]/[仓库名]/[分支名]

但是使用这种方式访问图片很慢，所以教程中使用了jsdelivr作为cdn加速。改变域名即可，不需要任何其他配置。

jsdelivr的DNS污染任然没有解决，这里不妨尝试fastly.jsdelivr.net（只做实验）

vscode picgo插件设置

点击picgo插件右下方设置，参考之前配置PicGo内容，将 Picgo>Pic Bed:Current 和 Picgo>Pic Bed>Github:xxx 部分设置完成。

使用

参考我之前的文章 picgo-sm-ms-v2-图床搭建

使用picgo手动上传图片和vscode快捷键上传图片。此外，PicGo客户端 还有相册功能，可以对已上传的图片进行删除，修改链接等快捷操作，PicGo 还可以生成不同格式的链接、支持批量上传、快捷键上传、自定义链接格式、上传前重命名等。

至此，picgo+github图床，以及vscode的快速上传环境全部搭建完毕。

结束之前

在其它博客发现下面这段提示，目前并未遇到这样的问题，但记录在案，待验证。

如果你配置了jsDelivr 加速访问，上传成功后，你会发现图片都显现不了，按照 jsDelivr官方访问格式可以看出，使用jsDelivr访问是需要GitHub发布一个版本的，所以我们需要在将图床仓库发布一个版本，然后才能访问。官方推荐访问格式如下：

https://cdn.jsdelivr.net/gh/user/repo@version/file
