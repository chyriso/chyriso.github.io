picgo+sm.ms v2 图床搭建

Posted on 2022-06-03 Edited on 2022-06-15 In major , website , pic_bed Views: Comments: Symbols count in article: 1.9k Reading time ≈ 2 mins.

说明

PicGo

一个用于快速上传图片并获取图片 URL 链接的工具，可以在下面地方了解它。

GitHub项目地址

Getting-Started Doc

PicGo 本体支持图床：七牛图床 腾讯云 COS 又拍云 GitHub SM.MS V2 阿里云 OSS Imgur

下载地址

Molunerfinn PicGo 个人仓库

山东大学镜像站

sm.ms

这是一个在线图床，可以免费提供5GB空间的永久存储空间，图片链接支持https，可以删除上传的图片，提供多种图片链接格式。每个图片最大5M，单次最多上传10张。

官网地址：https://sm.ms

picgo-plugin-smms-user

为 PicGo 开发的一款插件，新增了SM.MS注册用户图床。使用SM.MS V2的API上传，适用于注册了SM.MS的用户。填写Authorization即可。

当前 PicGo 版本 v2.3.1 ，默认支持 sm.ms 版本为 v1，而 sm.ms 官方支持的是 v2 版本的API， 所以需要在 PicGo 中使用该插件。

gitub地址：https://github.com/xlzy520/picgo-plugin-smms-user

vscode

这是一个 markdown 编辑工具，为它安装picgo插件并对插件设置，就可以通过快捷键快速上传图片到sm.ms 的存储空间，并在本地生成markdown语法的图片引用。其他 markdown 编辑器比如 typora 也应当有类似功能。你可以从官网了解更多信息。

下载地址：https://code.visualstudio.com/download

搭建

安装

安装 PicGo，请确保你安装了 Node.js， 并且版本 >= 8。

安装 picgo-plugin-smms-user 插件：

  • 在线安装

    打开 PicGo 窗口，选择插件设置，搜索smms-user安装，然后重启应用即可。

  • 离线安装

    克隆该项目

    复制项目到 picgo 根目录

    Windows: %APPDATA%\picgo\

    Linux: $XDG_CONFIG_HOME/picgo/ or ~/.config/picgo/

    macOS: ~/Library/Application\ Support/picgo/

    切换到新目录执行 (或者在picgo插件设置中点击导入插件)

    npm install ./picgo-plugin-smms-user

    然后重启应用即可。

安装 vscode：

  • User Installer版：会安装在当前计算机帐户目录,意味着如果使用另一个帐号登陆计算机将无法使用别人安装的vscode。

  • System Installer版：安装在非用户目录,自定义安装位置,任何帐户都可以使用。

根据需求自行从官网地址安装。

设置

通用设置

在sm.ms注册账号，注册完成后在Dasboard中选择 API Token，点击生成一个 Secret Token。

打开Picgo软件，点击图床设置，选择SM.MS-登录用户，将之前生成的 Secret Token 填入Auth 中，确定。

vscode用户

  • 配置

    在vscode安装插件 “picGo” ，点击插件右下角“扩展设置”，为用户设置以下三项：

    Picgo>Pic Bed:Current - 选择 smms

    Picgo>Pic Bed>Smms:Token - 写入在 sm.ms 官网生成的 Secret Token 。

    Picgo>Pic Bed:Uploader - 选择 smms

  • Keyboard shortcuts

    You can change all the shortcuts below as you wish.

    OSUploading an image from clipboardUploading images from explorerUploading an image from input box
    Windows/UnixCtrl + Alt + UCtrl + Alt + ECtrl + Alt + O
    OsXCmd + Opt + UCmd + Opt + ECmd + Opt + O
